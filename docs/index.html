Intelligence over convenience.

Voice Biometrics that stays in your house.

VoiceBM identifies who is speaking and publishes identity signals via MQTT for Home Assistant workflows. No cloud dependency required. Built like an engine, not a toy.

Room context is for collection and routing. Identity is global. Those two worlds don’t mix.

Features

Practical features for real homes — fast ID, clean controls, and automation-friendly outputs.

Speaker identification engine
Create embeddings, match to gallery, fall back to a virtual user identity for unknown speakers.
Home Assistant friendly
MQTT-first architecture so identities, sensors, and controls can show up as native entities.
Blocklist contract
Per-identity block switches (including user) so STT can fail silently when blocked.
Room scalability
Room names are engineering context for routing and organization — identity remains global.
